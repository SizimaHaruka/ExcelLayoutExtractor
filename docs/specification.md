# 仕様書

## 1. 概要

Excel テンプレートを解析し、ページ情報とレイアウト要素群を含む中間レイアウト JSON を生成する。

生成された JSON は、後続のプレビュー、手修正、HTML/PDF レンダリングに利用される。

## 2. システム構成

### 2.1 処理フロー

1. Excel ファイル読み込み
2. シート選択と印刷設定取得
3. セル情報、行高、列幅、罫線、マージ、画像の抽出
4. 座標変換
5. 線分抽出
6. 矩形候補抽出
7. テキスト抽出
8. 変数抽出
9. 画像抽出
10. 要素統合
11. JSON 出力
12. プレビュー生成

### 2.2 論理コンポーネント

- `ExcelParser`
- `CoordinateConverter`
- `LineExtractor`
- `RectDetector`
- `TextExtractor`
- `VariableExtractor`
- `ImageExtractor`
- `LayoutAssembler`
- `PreviewRenderer`

## 3. 入力仕様

### 3.1 受け付けるファイル

- `.xlsx`

`.xlsm` と `.xls` は対象外とする。

### 3.2 Excel テンプレート制約

- 可変値は `{{...}}` 記法で表現する
- 罫線ベースのレイアウトを対象とする
- セルの結合が多いレイアウトは精度が落ちる
- 図形や複雑なオブジェクトが存在すると精度が落ちる

### 3.3 取得対象情報

- セル値
- セル表示文字列
- フォント
- 配置
- 罫線
- 行高
- 列幅
- 結合セル情報
- 印刷範囲
- ページサイズ
- 余白
- 画像オブジェクト

## 4. 出力仕様

### 4.1 出力形式

UTF-8 エンコードの JSON ファイルを出力する。

### 4.2 JSON ルート構造

```json
{
  "page": {
    "width": 210,
    "height": 297,
    "unit": "mm"
  },
  "elements": []
}
```

### 4.3 page オブジェクト

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| width | number | Yes | ページ幅 |
| height | number | Yes | ページ高 |
| unit | string | Yes | 初期値は `mm` |
| marginTop | number | No | 上余白 |
| marginRight | number | No | 右余白 |
| marginBottom | number | No | 下余白 |
| marginLeft | number | No | 左余白 |

### 4.4 elements 配列

要素は以下のいずれかの型を取る。

- `line`
- `rect`
- `text`
- `variable`
- `image`
- `table`

### 4.5 要素仕様

#### 4.5.1 line

```json
{
  "type": "line",
  "x1": 10,
  "y1": 20,
  "x2": 100,
  "y2": 20
}
```

#### 4.5.2 rect

```json
{
  "type": "rect",
  "x": 10,
  "y": 30,
  "width": 180,
  "height": 60
}
```

#### 4.5.3 text

```json
{
  "type": "text",
  "x": 12,
  "y": 15,
  "width": 30,
  "height": 8,
  "text": "注文書",
  "style": {
    "fontFamily": "MS PGothic",
    "fontSize": 12,
    "fontWeight": "bold",
    "textAlign": "center"
  }
}
```

#### 4.5.4 variable

```json
{
  "type": "variable",
  "x": 80,
  "y": 30,
  "width": 40,
  "height": 8,
  "key": "customer_name",
  "format": null
}
```

#### 4.5.5 image

```json
{
  "type": "image",
  "x": 120,
  "y": 40,
  "width": 30,
  "height": 20,
  "source": "images/logo_001.png"
}
```

#### 4.5.6 table

`table` は後続フェーズで利用する拡張要素とし、MVP では必須出力としない。

## 5. 変数仕様

### 5.1 基本文法

- `{{key}}`
- `{{detail[].item}}`
- `{{amount|#,##0}}`
- `{{date|yyyy-MM-dd}}`

### 5.2 抽出ルール

- `{{` と `}}` に囲まれた文字列を変数候補とみなす
- `|` が含まれる場合は、前半をキー、後半を書式とみなす
- 不正な構文はテキストとして残し、警告対象とする

## 6. 座標仕様

### 6.1 単位

内部正規化単位は `mm` とする。

### 6.2 座標原点

- ページ左上を `(0, 0)` とする
- 印刷範囲を基準に要素位置を算出する

### 6.3 変換対象

- 列幅を絶対幅へ変換する
- 行高を絶対高へ変換する
- 累積値からセルの絶対座標を求める

## 7. 線分抽出仕様

### 7.1 入力

- セルごとの上下左右の罫線情報

### 7.2 処理

- セル境界をエッジとして抽出する
- 同一直線上にあり、かつ接続可能な線分をマージする
- 線種や太さが異なる場合は別線分として扱う

### 7.3 出力

- 正規化済みの `line` 要素配列

## 8. 矩形抽出仕様

### 8.1 初期方針

- 罫線による閉領域を矩形候補として検出する
- MVP では自動精度よりも安定出力を優先する

### 8.2 備考

- 小矩形群は将来的に `table` 候補とみなす

## 9. テキスト抽出仕様

### 9.1 対象

- セル文字列
- 固定見出し
- ラベル
- 変数テンプレート以外の文字列

### 9.2 付与属性

- 位置
- 領域
- フォント
- サイズ
- 太字
- 配置

## 10. 画像抽出仕様

### 10.1 対象

- Excel 内の埋め込み画像

### 10.2 出力

- 画像領域
- 参照パスまたは一意 ID

## 11. プレビュー仕様

### 11.1 MVP

- HTML ベースの簡易プレビュー
- JSON 要素を可視化できること
- line, rect, text, variable, image を表示できること

### 11.2 後続

- PDF 出力
- 修正 UI との連携

## 12. エラー仕様

- 解析不能なファイルは処理失敗として返す
- 未対応要素は無視せず警告として収集する
- 変数構文エラーは一覧化できるようにする
- ログに対象シート名とセル範囲を残す
