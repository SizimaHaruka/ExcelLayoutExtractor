# 要件書

## 1. 文書の目的

本書は、Excel 帳票からレイアウトを抽出し、帳票実装に再利用可能な中間レイアウト JSON を生成するシステムの業務要件、対象範囲、成功条件を定義する。

本書は、プロダクトの判断軸を統一し、設計・実装・検証のぶれを抑えることを目的とする。

## 2. 背景

既存業務では、Excel で設計された請求書、注文書、検査表などを Web または PDF 帳票として再実装する際に、レイアウトの目視確認と手作業によるトレースに多くの工数が発生している。

本プロジェクトは、Excel を帳票設計の入力手段として活用しつつ、そのレイアウト情報を機械可読な JSON に変換し、後続のレンダリングや手修正に接続できる状態を作る。

## 3. 目的

- 既存の Excel 帳票からレイアウト構造を抽出する
- 帳票実装で再利用可能な中間レイアウト JSON を生成する
- 帳票移行に必要な初期トレース工数を削減する
- 変数領域を抽出し、動的データ差し込みの起点を作る
- 人手による最終調整を前提に、再実装作業を短縮する

## 4. 非目的

- Excel 帳票を完全自動で再現すること
- Excel と互換なレンダリングエンジンを実装すること
- すべての Excel 機能を解釈すること
- Excel の計算結果を完全に移植すること
- 初期段階から高度な帳票推定や自動補正を完成させること

## 5. スコープ

### 5.1 対象

- 定型帳票
- 単票帳票
- 単純な明細帳票
- 請求書、注文書、納品書、検査表などの帳票
- セルベースでレイアウトされた Excel テンプレート

### 5.2 初期対象外

- 高度な条件付き書式
- 複雑なページ分岐
- 高度なグラフ、図形、SmartArt を含む帳票
- Excel の関数計算ロジックに強く依存する帳票
- 複雑な複票レイアウト
- Excel のマクロや外部参照に依存する帳票

## 6. 利用シナリオ

### 6.1 主シナリオ

1. 利用者が Excel テンプレートを入力する
2. システムがレイアウト情報を抽出する
3. システムが中間レイアウト JSON を生成する
4. 利用者がプレビューで確認する
5. 必要に応じて手修正し、帳票実装に利用する

### 6.2 期待される効果

- 新規帳票実装の初期工数削減
- 既存帳票の構造可視化
- Excel とレンダラの責務分離
- 帳票定義の再利用性向上

## 7. 入力要件

### 7.1 Excel テンプレート要件

- 可変値は `{{key}}` 形式で明示する
- 配列要素は `{{detail[].item}}` 形式を許可する
- 書式付き変数は `{{amount|#,##0}}` のような形式を許可する
- 大量の merge 乱用は避ける
- レイアウト表現は図形よりセル罫線を優先する
- 不要な装飾は抑制する
- できるだけフォントや表現を統一する

### 7.2 印刷設定要件

- ページサイズを取得可能であること
- 余白情報を取得可能であること
- 印刷範囲を取得可能であること

## 8. 出力要件

### 8.1 必須出力

- ページ情報
- 線分要素
- 矩形要素
- 固定文字要素
- 変数要素
- 画像要素
- JSON ファイルとしての永続化

### 8.2 出力の位置づけ

- 出力 JSON は帳票実装用の正本とする
- HTML または PDF は JSON から生成される派生物とする

## 9. 品質要件

- 帳票レイアウトの 80% 程度を自動再現できること
- 変数領域が大きく破綻せず抽出されること
- 手修正で完成可能な状態まで持っていけること
- 同一入力に対して安定した JSON を生成すること
- エラー時に原因を追跡しやすいこと

## 10. 想定失敗ケース

- merge 多用により構造が崩れる
- 罫線が不連続で線抽出が破綻する
- 文字位置合わせに依存していて構造推定が難しい
- 表と外枠を誤認識する
- 線分を過剰結合または過少結合する
- 画像領域を誤判定する
- フォント差異や改ページ差異が生じる

## 11. 成功条件

- Excel 帳票から主要レイアウト情報を抽出できる
- 抽出結果を JSON として後続工程に引き渡せる
- 変数領域を人が識別可能な精度で抽出できる
- 帳票再実装に必要な初期工数を削減できる

## 12. MVP 範囲

### 12.1 必須

- Excel 読み込み
- 座標変換
- 線抽出
- テキスト抽出
- 変数抽出
- JSON 出力
- 簡易プレビュー

### 12.2 後回し

- 矩形自動統合
- table 推定
- 高精度画像判定
- UI エディタ
- 高度な補正支援

## 13. 設計原則

- Excel は設計入力ツールである
- JSON は帳票定義の正本である
- レンダラは JSON を解釈して出力する
- 完全自動ではなく半自動トレースを前提とする
